## Defeitos Do Software ##

## 1.FICHA DE ANAMNESE
  A fase do ciclo em que o defeito deveria ter sido identificado acontece quando, ao realizar o cadastro, aparece a mensgaem "salva", mas ao pesquisar o nome do paciente,o cadastro não aparece.

  Os impactos para o paciente e para o negócio são significativos, pois como irá confirmar se o foi realmente cadastrado, se, ao pesquisar pelo nome, o cadastro não aparece. como iria se consultar?

Esse defeito deveria ter sido encontrado na fase de testes. quanto mais cedo achar um erro no software, melhor é o custo para corrig-los.




## 2. AGENDA DE CONSULTAS##

 o defeito é identificado quando o sistema agendeu duas consultas no mesmo horário.

Os impactos para o paciente e o negócio são significativos,pois o paciente irá ser atendido em um horário em que  ja existe outro paciente agendado. isso  gera conflitos desnecessário para o negócio

Esse defeito deveria ter sido prestado mais atenção na fases de testes, para evitar esse tipo de situação.





## 3. CADASTRO DO  PSÍCOLOGO

 O principal problema ocorre a partir do momento em que o sistema valida um cadastro inválido

   O impacto é que o sistema está aceitando normalmente um cadastro inválido,o que não deveria acontecer. isso gera conflitos desnecessários para o negócio e pode causar ate uma falta de confiança no sistema.

   Esse defeito deveria ter sido idenficado na fases de testes. quanto mais cedo esse erro fosse encontrado, menor seria o custo para corrigi-lo e menores seriam os impactos para o negócio.



   ## 4.CONTROLE FINANCEIRO
    O defeito se encontra quando o sistema aceitou normalmente um valor negativo, embora o esperado fosse impedir valore esse tipo de valor.

    os impactos são evidente pois o saldo financeiro ficará incorreto. podendo gerar confusão na contabilidade do sistema e comprometer a confiabilidade das informações financeiras.

    Quanto mais cedo encontrar esse erro menor será para corrigi-lo 

## 5. DEFEITO NO LOGIN
 O defeito se encontra quando, após varias tentativas incorretas de senhas, o sistema  continua permitindo  novas tentativas.

 o impacto acontece quando o resultado esperado não acontece e o sistema permanece sem qualquer proteção. isso pode gerar problema de segurança alem de causar falta de confiança no sistema.

 esse defeito poderia ter sido evitado se tivesse sido identificado na fase de testes. Assim, seria possivel evitar problema de segurança e aumentar a confiança no sistema.
